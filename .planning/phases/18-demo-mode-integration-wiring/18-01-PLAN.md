    - Error handling: if fetch fails, set error state. No retry (data is static)
  
  - **Advance effect** — when `isActive && timeline && isPlaying && currentIndex < timeline.length`:
    - Use setInterval(advanceIntervalMs) — increment currentIndex
    - Clear interval in cleanup
    - When currentIndex >= timeline.length - 1: setIsPlaying(false) (timeline complete)
  
  - **Write + deltas effect** — when `currentIndex` changes and timeline[currentIndex] exists:
    - Extract event = timeline[currentIndex]
    - Call `setMatch({ score: event.score, phase: event.phase, minute: event.minute, homeTeam: "Team A", awayTeam: "Team B" })`

        - Use `baseInput = state.v1ZoneData ?? presets.normal` (always apply from base, not accumulated — prevents stacking per Research Pitfall #2)
        - `adjusted = applyPhaseTransitionDeltas(baseInput, event.eventType, event.zoneDeltas)`
        - Call `state.initializeSim(adjusted)` — this re-runs the simulation with adjusted inputs
      - Goal events (minute 23, 67, 82): trigger per-zone arrival surges
      - Halftime event (minute 45): triggers all-zone capacity -10%
      - Full-time event (minute 90): triggers all-zone demand -80%
  
  - **Visibility effect** — pause/resume interval based on document.hidden (same pattern as useMatchPoller)
  
  - **Cleanup** — clear interval on unmount via return cleanup
  
  - Return `{ currentEvent: timeline?.[currentIndex] ?? null, isPlaying, totalEvents: timeline?.length ?? 0, currentIndex }`
